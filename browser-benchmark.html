React Position Update Benchmark (Real Browser)

Measures both commit cost and frame pacing in Chromium with multiple element counts.

Scenarios: 100, 500, 1000 moving elements. Each frame updates every element.

What This Script Does

  • Creates a React scene with many moving dots.
  • Runs the same motion workload with three different React update patterns.
  • Tests three sizes: 100, 500, and 1000 elements.
  • Warms up first, then measures many frames and multiple updates per frame.
  • Records average React commit cost, frame pacing, and jank rate.

Measured Variants

// A: inline style prop
function DotA({ x, y }) {
  return <div className="dot" style={{ transform: `translate(${x}px, ${y}px)` }} />
}

// B: ref + useLayoutEffect
function DotB({ x, y }) {
  const ref = useRef(null)
  useLayoutEffect(() => {
    const elm = ref.current
    if (!elm) return
    elm.style.transform = `translate(${x}px, ${y}px)`
  }, [x, y])
  return <div className="dot" ref={ref} />
}

// C: callback ref
function DotC({ x, y }) {
  return (
    <div
      className="dot"
      ref={(elm) => {
        if (elm) elm.style.transform = `translate(${x}px, ${y}px)`
      }}
    />
  )
}

How To Read Results

  • Mean commit: average time spent doing the React update work.
  • P95 commit: slow tail for update cost.
  • Mean/P95 frame: pacing between animation frames.
  • Frames >16.7ms: share of frames missing a 60fps budget.
  • Lower numbers are better across all columns.
